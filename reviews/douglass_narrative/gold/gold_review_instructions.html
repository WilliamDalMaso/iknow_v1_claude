Douglass Gold Review Instructions

Date: 2026-05-27

Gold rows are the answer key. Do not mark a row authoritative unless the rendered page, overlay boxes, object card, source lines, and candidate bucket all support the decision.

Review Surface

  1. Open data/runs/douglass_narrative/phase1_v3/phase1_audit.html.
  2. Use the page image, bounding-box overlays, object cards, source line ids, and candidate buckets together.
  3. Inspect the sampled pages listed in gold_pages.json.
  4. Prefer the visual PDF page over clean-looking extracted text when deciding boundaries.

Paragraph Boundary Rows

Add rows to gold_paragraph_boundaries.jsonl only when the source lines are known to form one correct paragraph. Use review_status: "authoritative" only after review. Use review_status: "uncertain" if the visual evidence is ambiguous.

Object Label Rows

Add rows to gold_object_labels.jsonl for selected objects using one of: main_paragraph, structure, metadata, page_artifact, or unknown. If unsure, use unknown and explain why in reason.

Rules

  1. Do not guess missing text.
  2. Do not mark seed examples authoritative without visual inspection.
  3. Record uncertainty explicitly instead of forcing a label.
  4. No merge policy can become active unless it improves against authoritative gold rows and does not worsen audit safety metrics.
