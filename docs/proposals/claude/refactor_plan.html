Claude Lane — Refactor Plan

Author: Claude · Date: 2026-05-28

Proposal only. This plan must not be executed until explicitly requested. Any refactor is behavior-preserving by definition: python3 -m pytest tests must stay green and a re-run of Douglass v3 must produce byte-identical artifacts (or a documented, reviewed diff). Refactoring is not an extraction-behavior change.

1. Problem

src/phase1_extract.py is 7,524 lines with 106 top-level functions spanning IO, segmentation, furniture detection, cross-page joins, promotion, review reporting, gold evaluation, bbox/span diagnosis, and HTML rendering. This concentration makes review costly, raises regression risk per edit, and couples unrelated concerns.

2. Guard Rails (must hold before, during, after)

  • [DONE] The golden-output regression test exists: tests/test_phase1_regression.py re-runs Douglass and asserts it reproduces the committed normalized fingerprint (docs/baselines/…/normalized_manifest.json). Verified deterministic across two independent runs. It skips cleanly where the PDF is absent. Decomposition is now safe to begin.
  • Pure relocation only — no logic edits in the same commit as a move.
  • One concern per commit; tests green at every step.
  • Preserve the public entry points and all artifact filenames/paths.

2a. Progress (2026-05-28)

Decomposition is underway, one behavior-preserving step per commit, each verified by the regression test (every step reproduced the Douglass fingerprint exactly). Monolith 7,524 → 7,267 lines so far. Extracted modules:

  • phase1/io_utils.py — IO + formatting helpers.
  • phase1/text_utils.py — text-pattern constants + pure text/line helpers.
  • phase1/layout_geometry.py — line margin/indent/height/gap helpers.
  • phase1/heuristics.py — paragraph-text + cross-page provenance heuristics.
  • phase1/span_metrics.py — bbox/span range + severity bucketing.

Remaining (larger / more entangled, reference module-level policy constants and call many siblings): scattered pure helpers (page_status, confidence_for_paragraph, page/bbox helpers), then furniture detection, cross-page joins, promotion, review, gold evaluation, and the ~1,900-line audit-HTML renderer.

3. Proposed Module Seams

Decompose into a src/phase1/ package, preserving a thin phase1_extract.py facade that re-exports the existing names (so tests and any callers keep working):

  • phase1/io_utils.py — write_json, read_jsonl, sha256_file, path helpers, commented-jsonl readers.
  • phase1/layout.py — line normalization, margins, paragraph segmentation, classification.
  • phase1/furniture.py — running-furniture detection, page-number tokens, furniture profiles.
  • phase1/cross_page.py — cross-page + chained continuation experiments and join logic.
  • phase1/promotion.py — promotion blockers and canonical paragraph artifacts.
  • phase1/review.py — canonical review, merge experiments, failure taxonomy, bbox/span diagnostics.
  • phase1/gold.py — gold evaluation and authoritative-row logic.
  • phase1/audit_html.py — phase1_audit.html and overlay rendering.
  • phase1/pipeline.py — orchestration / run entry point.

4. Sequence

  1. [DONE] Capture & commit a golden fingerprint + regression test.
  2. Extract io_utils (lowest risk, no domain logic).
  3. Extract audit_html (large, self-contained rendering).
  4. Extract gold and review (read-only reporting; cannot change extraction).
  5. Extract furniture, then layout, then cross_page, then promotion.
  6. Introduce pipeline.py; reduce phase1_extract.py to a facade.
  7. Re-run Douglass v3; confirm artifact hashes match the manifest.

5. Test Suite Split (parallel, optional)

tests/test_phase1_extract.py (48KB) should mirror the module split into tests/phase1/test_layout.py, test_cross_page.py, etc. Do this only after the source split stabilizes, to avoid double churn.

6. Explicitly Out of Scope

No detector tuning, no policy changes, no new artifacts, no OCR/model/retrieval/graph work. This plan only reshapes existing code. Behavior change requires its own gold-gated, side-effect-reviewed proposal.
