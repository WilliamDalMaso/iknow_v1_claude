Claude Lane — Observability Upgrade Plan

Author: Claude · Date: 2026-05-28

Largely delivered (2026-05-28). The Claude lane now runs its own dashboard at http://127.0.0.1:8799 (src/claude_observability_server.py), distinct from the legacy/Codex dashboard on 8765 so both run at once. The writer (src/observe.py) emits the structured schema below. Remaining: decision-artifact linkage (item 4) once the Phase B refactor exposes a clean pipeline seam.

1. Current State

  • observe.py appends {timestamp, kind, message, details} events to data/runs/observability/events.jsonl.
  • observability_server.py serves a dependency-free dashboard on port 8765.
  • The event log is git-ignored and machine-local — the record of why/when decisions were made is not durable or reproducible.

2. Gaps

  • No run correlation. Events are a flat stream; they are not tied to a book_id + phase1_vN + git commit, so you cannot reconstruct "which run produced these numbers."
  • No schema/versioning. Free-form kind means typos (codex vs claude) fragment the stream silently.
  • No durable decision ledger. Adoption decisions live in run artifacts, not in observability; the two are not linked.
  • No second-book dimension. Nothing namespaces events by book.

3. Upgrades

  1. [DONE] Structured event fields: schema, level, actor, book_id, run_id, phase, git_commit added to src/observe.py. Backward compatible — legacy flat events still parse and render.
  2. [DONE] Auto git-commit stamp: every event records the short commit automatically; --book/--run/--phase correlate events to a run.
  3. [DONE] Dashboard filters & UX: severity colors, search, faceted filters (level/actor/book/phase), issues-only, group-by-run, pause, keyboard shortcuts, summary cards — src/claude_observability_server.py on port 8799.
  4. [PENDING] Decision linkage: emit a paired event when an adoption-decision artifact is written, so the dashboard indexes the durable ledger. Deferred until the Phase B refactor exposes a clean pipeline seam.
  5. [DONE] Schema/helper tests: tests/test_claude_observability.py asserts the writer schema, normalization of legacy events, and aggregate summaries.

4. Sequencing vs. Refactor

Items 1, 2, and 5 are isolated additions to observe.py and can land independently of the monolith refactor. Items 3 and 4 touch the server and the pipeline; defer item 4 until the refactor exposes a clean pipeline.py seam to hook into.

5. Out of Scope

No external telemetry services, no new ports, no network exposure beyond localhost, no model calls.
