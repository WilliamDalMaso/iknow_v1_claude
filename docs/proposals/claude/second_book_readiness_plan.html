Claude Lane — Second Book Readiness Plan

Author: Claude · Date: 2026-05-28

The hard rule stands: do not widen to many books before one book passes Phase 1 gates. Douglass is safe_for_downstream: false, so a second book must not be adopted or promoted. This plan is about readiness, not activation.

1. Observation

data/books/the_hobbit.pdf is already present (git-ignored). It is the natural second book candidate. Nothing has been run against it, and it must stay that way for adoption until Douglass clears its gates.

2. Why A Second Book Is Risky Now

  • Single-book calibration. Margins, furniture patterns, and continuation heuristics were tuned against Douglass. The Hobbit's layout (chapter ornaments, verse, dialogue, illustrations) will exercise untested paths.
  • No gold set exists for book #2. The gold-gated method requires authoritative gold rows before any policy can be judged. Without them, warning counts are not truth.
  • Book-specific decisions are not generalized. reviews/douglass_narrative/ join and override decisions do not transfer.
  • Hard-coded Douglass facts. Governance prose cites Douglass page numbers (e.g. 59–61, page 109) and counts; reusing that machinery blindly risks confusion.
  • Untracked baseline. Because runs are git-ignored, a second book's output is equally unreproducible until the artifact-tracking gap (R1) is resolved.

3. Readiness Checklist (before book #2 adoption is even considered)

  1. Douglass reaches safe_for_downstream: true (resolve the 23 likely-true grouping defects and 39 bbox/span cases through the documented remediation path).
  2. The artifact-reproducibility gap (R1) is closed so any book's run can be verified from a clean clone.
  3. The monolith is decomposed enough that book-specific calibration is isolated from generic logic.
  4. A gold-set authoring workflow is repeatable for a new book (instructions + seed + authoritative promotion).
  5. A diagnostic dry run on The Hobbit (extraction + audit only, no promotion, no adoption) to surface where single-book assumptions break — explicitly labeled non-canonical.

4. Recommended First Step (when unblocked)

A read-only diagnostic pass: run Phase 1 candidate extraction on The Hobbit, generate the audit HTML and page images, and write a "single-book-assumption breakage" report — without creating gold sets, promoting paragraphs, or adopting any policy. This is the lowest-risk way to learn generalization gaps while keeping every gate intact.

5. What Stays Blocked

Until step 1–4 above are done: no second-book promotion, no shared-policy adoption, no cross-book retrieval/graph work. The Hobbit stays a candidate, not a canonical book.
