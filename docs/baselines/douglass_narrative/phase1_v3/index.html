Douglass phase1_v3 — Reproducibility Baseline

Captured by Claude: 2026-05-28 · Phase: A (make it trustworthy)

This closes risk R1 from the consistency report. Phase 1 run output lives under git-ignored data/runs/, so the documented canonical state was previously unverifiable from a clean clone. This baseline commits the evidence needed to verify a re-run, without committing raw PDFs or bulky output.

Contents

  • artifact_manifest.json — byte-level sha256 + size for all 164 files of the original captured phase1_v3 run (Phase A snapshot of the Codex-produced run).
  • normalized_manifest.json — Phase B golden-output fingerprint: sha256 of each artifact's run-invariant content (timestamps, run_id, and absolute paths neutralized; volatile JSON keys stripped). This is what a re-run reproduces, and what tests/test_phase1_regression.py checks.
  • reports/ — tracked snapshot of the small decision-bearing report JSONs that back the governance numbers, including post_adoption_canonical_safety_report.json (warning_count: 129, safe_for_downstream: false), validation_report.json, canonical_promotion_report.json, gold_evaluation_report.json, and the two policy adoption decisions.

Determinism findings (Phase B)

Two independent re-runs of the current code reproduce each other exactly, and reproduce this normalized baseline (MATCH). The pipeline is deterministic given the same PDF. Two facts about the original captured run:

  • All JSON reports reproduce identically — the governance numbers are fully reproducible from current code.
  • The original run dir contains a stale review_overrides.jsonl that a current run does not produce (it differs from the curated reviews/ source), and its phase1_audit.html rendering differs slightly from current code. These are noted in the consistency risk report and do not affect canonical numbers.

How To Verify A Re-run

Byte-identity of the original captured run (Phase A):

python3 src/baseline_manifest.py verify \
  data/runs/douglass_narrative/phase1_v3 \
  docs/baselines/douglass_narrative/phase1_v3/artifact_manifest.json

Run-invariant reproduction of a fresh run (Phase B, the one that matters for refactors):

python3 src/phase1_extract.py data/books/douglass_narrative.pdf \
  --book-id douglass_narrative --run-id phase1_v3_check
python3 src/baseline_manifest.py verify-fingerprint \
  data/runs/douglass_narrative/phase1_v3_check \
  docs/baselines/douglass_narrative/phase1_v3/normalized_manifest.json

MATCH means the run reproduces the baseline. Any MISSING, UNEXPECTED, or CHANGED line is a drift signal. The same verify-fingerprint path is exercised automatically by tests/test_phase1_regression.py. The tool only hashes content; it does not change extraction behavior.

Scope & Limits

  • This is a byte-level baseline of the current local run. It establishes a fixed point; it does not claim the run is flawless. safe_for_downstream remains false.
  • Re-run determinism itself is validated in Phase B via a golden-output regression test that calls this same verify path.
