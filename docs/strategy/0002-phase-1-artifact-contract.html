Phase 1 Artifact Contract

Date: 2026-05-27

Phase 1 produces an auditable document package. It does not produce embeddings, retrieval chunks, claims, entities, relations, graphs, or user-facing answers.

Run Location

Generated Phase 1 runs live under data/runs/<book_id>/phase1_v0/. Runtime run artifacts are ignored by git unless a curated finding is promoted into docs/.

Required Artifacts

ArtifactPurposeGate Role
source_manifest.json Records source path, file size, SHA-256 hash, run id, and extraction tool metadata. Proves which PDF was processed.
page_inventory.jsonl One row per PDF page with dimensions, raw text counts, image/table counts, status, and review flags. Proves every page is accounted for.
raw_pages.jsonl Raw extracted text preserved page by page before cleanup. Prevents silent rewriting.
layout_objects.jsonl Initial deterministic page objects, usually line-level in v0, with type guesses and confidence. Begins object-level organization before retrieval.
clean_objects.jsonl Cleaned object text with cleanup operations attached. Makes cleanup inspectable.
canonical_reading_order.json Ordered object ids plus summary counts and review flags. Defines the first canonical reading sequence.
cleanup_log.jsonl Append-only log of each cleanup operation applied to extracted text. Prevents untracked text mutation.
phase1_audit.html Human-readable local audit page summarizing status, risks, and page samples. Primary v0 review surface.

V0 Acceptance

  1. The page inventory row count equals the PDF page count.
  2. Image-only, blank, mixed, and text pages are explicitly labeled.
  3. Raw extracted text is preserved unchanged.
  4. Known extraction noise such as (cid:...) is flagged for review.
  5. Cleanup operations are logged per object.
  6. The audit page exposes enough evidence to decide the next extraction improvement.

Known Limits

Phase 1 v0 is line-object based. It is allowed to be imperfect, but not opaque. Its job is to reveal extraction risks and establish the artifact spine before we improve object segmentation.
