Phase 1 Artifact Contract

Date: 2026-05-27

Phase 1 produces an auditable document package. It does not produce embeddings, retrieval chunks, claims, entities, relations, graphs, or user-facing answers.
Phase 1 is candidate-first. It does not call extracted text canonical until an audit step has promoted it.

Run Location

Generated Phase 1 runs live under data/runs/<book_id>/phase1_v*/. Runtime run artifacts are ignored by git unless a curated finding is promoted into docs/.

page_artifacts_candidates.jsonl Likely page numbers, repeated headers, running footers, and other page artifacts with subtype, confidence, warnings, source ids, source lines, bounding boxes, and furniture evidence. unknown_objects.jsonl Objects that cannot be safely classified yet. Prevents silent deletion and false confidence.

page_images/ Generated rendered JPEG image for each PDF page, named page_0001.jpg and so on. Provides visual evidence for audit without OCR, AI review, or canonical promotion.
reviews/<book_id>/review_overrides.jsonl Tracked curated reviewer corrections for candidate bucket assignment, one JSON object per line. Required fields: object_id, original_bucket, corrected_bucket, reason, reviewer, date, and evidence_reference. Prevents a policy from becoming active merely because it feels ready; adoption must be traceable to gate evidence and validation.

Current Acceptance

  1. The page inventory row count equals the PDF page count.
  2. Image-only, blank, mixed, and text pages are explicitly labeled.
  3. Raw extracted text is preserved unchanged.
  4. Known extraction noise such as (cid:...) is flagged for review.
  5. Cleanup operations are logged per object.
  6. Normal prose lines are merged into paragraph candidates, with source line ids preserved.
  7. Heading detections remain structure_candidate rows until audited.
  8. Every extracted object appears in exactly one candidate bucket unless duplicated only as a linked reference.
  9. No output is named canonical before audit promotion.
  10. Rendered page image count matches source page count.
  11. The audit page links each inspected page to its rendered page image.
  12. The audit page draws overlay boxes for objects with complete bounding boxes.
  13. The audit page can hide, show, filter, and focus overlay boxes without changing extraction outputs.
  14. The audit page generates copy-ready override JSONL templates for selected objects without applying them.
  15. Repeated page furniture is detected from deterministic repetition, position, and page-number evidence.
  16. Artifact false-positive risks are visible before any canonical exclusion is allowed.
  17. Review overrides are validated, evidence-bound, and candidate-only.
  18. Canonical paragraph promotion is paragraph-only and records promoted rows, blocked rows, and promotion counts.
  19. No structure, page artifact, or unknown object appears in canonical_paragraphs.jsonl.
  20. canonical_paragraph_review_report.json row counts match canonical_paragraphs.jsonl.
  21. The audit page shows canonical paragraph review counts, warning categories, risk clusters, corrective-action recommendations, samples, and safe_for_downstream.
  22. The audit page lets a reviewer compare raw extracted objects against their candidate bucket assignments.
  23. gold_evaluation_report.json reports gold row counts, excludes non-authoritative seed rows from scoring, and states whether the gold set is sufficient to judge merge-policy adoption.
  24. No paragraph merge policy can become active from heuristic warning improvement alone.
  25. cross_page_join_review_report.json exists when a cross-page continuation experiment proposes joins, and the audit page exposes those joins before adoption.
  26. Cross-page join decisions are validated against proposed join ids, pages, candidate ids, required fields, duplicate decisions, and valid decision values.
  27. The validation report passes.
  28. The audit page exposes enough evidence to decide the next extraction improvement.

Known Limits

The current Douglass run is still heuristic even though the audit surface is much stronger than the first pass. Paragraph purity, front matter boundaries, table handling, figure handling, captions, footnotes, lists, and section nesting are not solved yet. This is allowed only because the system preserves raw evidence, candidate buckets, page images, bounding boxes, review overrides, and validation instead of pretending the extraction is canonical.

Model Assistance Policy

No model is required for the first extraction pass. Phase 1 starts deterministic-first because clean extraction depends on visible evidence, not fluent interpretation. Models may be introduced only after raw/page/object artifacts exist, and only for bounded review tasks such as:

  1. reviewing flagged pages for likely extraction failure type,
  2. suggesting object classifications for ambiguous objects,
  3. checking whether a cleaned paragraph appears to preserve the raw text,
  4. proposing OCR repair for specific noisy spans, with human or deterministic verification.

A model must not silently rewrite the source text or become the extractor of record.
