Phase 1 Artifact Contract

Date: 2026-05-27

Phase 1 produces an auditable document package. It does not produce embeddings, retrieval chunks, claims, entities, relations, graphs, or user-facing answers.
Phase 1 is candidate-first. It does not call extracted text canonical until an audit step has promoted it.

Run Location

Generated Phase 1 runs live under data/runs/<book_id>/phase1_v*/. Runtime run artifacts are ignored by git unless a curated finding is promoted into docs/.

Required Artifacts

ArtifactPurposeGate Role
source_manifest.json Records source path, file size, SHA-256 hash, run id, and extraction tool metadata. Proves which PDF was processed.
page_inventory.jsonl One row per PDF page with dimensions, raw text counts, image/table counts, status, and review flags. Proves every page is accounted for.
raw_pages.jsonl Raw extracted text preserved page by page before cleanup. Prevents silent rewriting.
layout_objects.jsonl Initial deterministic page objects with type guesses, confidence, raw text, source line ids, and bounding boxes when available. Complete boxes include x0, x1, top, and bottom. Begins object-level organization before retrieval and provides visual overlay evidence.
clean_objects.jsonl Cleaned object text with cleanup operations attached. Makes cleanup inspectable.
main_paragraph_candidates.jsonl Body paragraph candidates with source object ids, source line ids, raw text, cleaned text, confidence, warnings, and bounding boxes when available. Separates likely main prose from structure and artifacts without calling it canonical.
structure_candidates.jsonl Meaningful non-body candidates such as titles, chapter headings, section markers, captions, or footnote-like objects. Preserves book structure evidence for later mapping and audit.
page_artifacts_candidates.jsonl Likely page numbers, repeated headers, running footers, and other page artifacts with subtype, confidence, warnings, source ids, source lines, bounding boxes, and furniture evidence. Prevents page artifacts from polluting paragraph retrieval while keeping them inspectable.
unknown_objects.jsonl Objects that cannot be safely classified yet. Prevents silent deletion and false confidence.
reconstruction_map_candidate.json Candidate map linking source objects to streams, paragraphs to pages, and non-body objects to their pages. Defines the provisional reconstruction without promoting it to canonical truth.
reading_order_candidate.json Ordered object ids plus candidate stream counts and review flags. Defines the provisional reading sequence for audit.
page_images/ Generated rendered JPEG image for each PDF page, named page_0001.jpg and so on. Provides visual evidence for audit without OCR, AI review, or canonical promotion.
reviews/<book_id>/review_overrides.jsonl Tracked curated reviewer corrections for candidate bucket assignment, one JSON object per line. Required fields: object_id, original_bucket, corrected_bucket, reason, reviewer, date, and evidence_reference. Preserves reviewed extraction decisions across fresh clones, machines, and regenerated runs.
review_overrides_applied.jsonl Generated record of curated review decisions replayed into a specific Phase 1 run. Lets review decisions change candidate streams without changing detector rules or promoting content to canonical.
canonical_paragraphs.jsonl Paragraph candidates promoted by the current evidence gate, with canonical paragraph id, source candidate id, raw text, clean text, source ids, bbox, promotion status, reasons, warnings, and applied override when present. Creates the first downstream-safe paragraph layer without promoting structure, artifacts, unknowns, OCR, AI review, retrieval, or graph work.
promotion_blockers.jsonl Rows for candidates not promoted by the paragraph gate, including blocker reasons and preserved evidence. Prevents silent non-promotion and makes canonical exclusions reviewable.
canonical_promotion_report.json Summary of paragraph promotion counts, blocked counts, warning counts, override-influenced promotions, and validation status. Proves the candidate-to-canonical gate was applied consistently.
canonical_paragraph_review_report.json Deterministic review of promoted canonical paragraphs for likely quality risks, with warning categories, severity, risk clusters, affected pages, risky samples, source candidate ids, page numbers, source lines, bounding boxes, likely corrective actions, and downstream safety recommendation. Prevents clean-looking canonical output from entering retrieval, embeddings, reasoning, or graph work without a reviewed risk signal.
cleanup_log.jsonl Append-only log of each cleanup operation applied to extracted text. Prevents untracked text mutation.
validation_report.json Checks required artifacts, page coverage, object ids, raw preservation, cleanup references, rendered page images, overlay data, and review override validity. Prevents silent artifact-contract drift.
phase1_audit.html Human-readable local audit page summarizing status, risks, page samples, page-by-page object inspections, repeated artifact patterns, false-positive risks, review filters, page images, overlay boxes, overlay controls, selected object details, and copy-ready review override templates. Primary review surface for deciding whether candidate streams and artifact exclusions are trustworthy, and for safely drafting manual curated overrides.

Current Acceptance

  1. The page inventory row count equals the PDF page count.
  2. Image-only, blank, mixed, and text pages are explicitly labeled.
  3. Raw extracted text is preserved unchanged.
  4. Known extraction noise such as (cid:...) is flagged for review.
  5. Cleanup operations are logged per object.
  6. Normal prose lines are merged into paragraph candidates, with source line ids preserved.
  7. Heading detections remain structure_candidate rows until audited.
  8. Every extracted object appears in exactly one candidate bucket unless duplicated only as a linked reference.
  9. No output is named canonical before audit promotion.
  10. Rendered page image count matches source page count.
  11. The audit page links each inspected page to its rendered page image.
  12. The audit page draws overlay boxes for objects with complete bounding boxes.
  13. The audit page can hide, show, filter, and focus overlay boxes without changing extraction outputs.
  14. The audit page generates copy-ready override JSONL templates for selected objects without applying them.
  15. Repeated page furniture is detected from deterministic repetition, position, and page-number evidence.
  16. Artifact false-positive risks are visible before any canonical exclusion is allowed.
  17. Review overrides are validated, evidence-bound, and candidate-only.
  18. Canonical paragraph promotion is paragraph-only and records promoted rows, blocked rows, and promotion counts.
  19. No structure, page artifact, or unknown object appears in canonical_paragraphs.jsonl.
  20. canonical_paragraph_review_report.json row counts match canonical_paragraphs.jsonl.
  21. The audit page shows canonical paragraph review counts, warning categories, risk clusters, corrective-action recommendations, samples, and safe_for_downstream.
  22. The audit page lets a reviewer compare raw extracted objects against their candidate bucket assignments.
  23. The validation report passes.
  24. The audit page exposes enough evidence to decide the next extraction improvement.

Known Limits

The current Douglass run is still heuristic even though the audit surface is much stronger than the first pass. Paragraph purity, front matter boundaries, table handling, figure handling, captions, footnotes, lists, and section nesting are not solved yet. This is allowed only because the system preserves raw evidence, candidate buckets, page images, bounding boxes, review overrides, and validation instead of pretending the extraction is canonical.

Model Assistance Policy

No model is required for the first extraction pass. Phase 1 starts deterministic-first because clean extraction depends on visible evidence, not fluent interpretation. Models may be introduced only after raw/page/object artifacts exist, and only for bounded review tasks such as:

  1. reviewing flagged pages for likely extraction failure type,
  2. suggesting object classifications for ambiguous objects,
  3. checking whether a cleaned paragraph appears to preserve the raw text,
  4. proposing OCR repair for specific noisy spans, with human or deterministic verification.

A model must not silently rewrite the source text or become the extractor of record.
