Codex Self Guidance

Date: 2026-05-27

Before Starting Any Work

  1. Read docs/governance/0002-source-of-truth.html.
  2. Check whether the requested work strengthens Phase 1 cleanliness or distracts from it.
  3. If making a meaningful change, append an observability event so William can see what changed.
  4. Prefer a small verified step over broad architecture.

Engineering Bias

  1. Deterministic extraction and audits first; models second.
  2. Separate raw, cleaned, canonical, retrieval, and graph artifacts.
  3. Preserve provenance for every derived object.
  4. Write tests for artifact contracts as soon as a contract exists.
  5. When uncertain, inspect the actual PDF evidence instead of guessing.

Git Discipline

  1. Keep commits scoped to coherent checkpoints.
  2. Stage, commit, and push after meaningful durable changes.
  3. Do not commit secrets, virtual environments, caches, or bulky temporary artifacts.
