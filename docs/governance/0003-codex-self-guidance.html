Codex Self Guidance

Date: 2026-05-27

Before Starting Any Work

  1. Read docs/governance/0002-source-of-truth.html.
  2. Read docs/governance/0005-codex-work-loop.html.
  3. Check whether the requested work strengthens Phase 1 cleanliness or distracts from it.
  4. If making a meaningful change, append an observability event so William can see what changed.
  5. Prefer a small verified step over broad architecture.

Engineering Bias

  1. Deterministic extraction and audits first; models second.
  2. Separate raw, cleaned, candidate, future canonical, retrieval, and graph artifacts.
  3. Do not call Phase 1 output canonical until it has passed audit, comparison, and review.
  4. Preserve provenance for every derived object.
  5. Write tests for artifact contracts as soon as a contract exists.
  6. When uncertain, inspect the actual PDF evidence instead of guessing.

Git Discipline

  1. Keep commits scoped to coherent checkpoints.
  2. Stage, commit, and push after meaningful durable changes.
  3. Do not commit secrets, virtual environments, caches, or bulky temporary artifacts.

GitHub Readme Exception

  1. README.md is allowed for GitHub reader experience.
  2. Do not move durable local documentation back to Markdown.
  3. When README.md and HTML docs disagree, update the HTML source of truth first.

Observability Discipline

  1. Use http://127.0.0.1:8765 as the single observability dashboard.
  2. Do not start alternate observability ports unless William explicitly asks.
  3. If the server needs a restart, stop the existing process and reuse port 8765.
