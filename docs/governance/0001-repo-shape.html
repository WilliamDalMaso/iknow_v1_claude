Repository Shape

Date: 2026-05-27

Initial Directories

  • docs/: HTML project memory, decisions, lessons, strategy, and history.
  • src/: reusable engine code only.
  • tests/: automated checks for extraction, cleanup, and artifact contracts.
  • data/books/: local source-book inputs and book-specific setup files.
  • data/runs/: generated run artifacts, grouped by book and run id.

Rules

  1. Generated book artifacts do not live beside source code.
  2. One run must be inspectable without reading unrelated book runs.
  3. Documentation is HTML, not Markdown.
  4. History and lessons are separated from strategy so future readers can distinguish what happened from what we intend.
  5. Phase 1 artifacts must show raw, cleaned, candidate, future canonical, and audit views separately.
