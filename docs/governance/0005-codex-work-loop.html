Codex Work Loop

Date: 2026-05-27

This is the self-reminder loop for future Codex work. Before building, Codex should re-orient around the source of truth, Phase 1 priority, and current lessons. After building, Codex should document what happened, what worked, what failed, and what should happen next.

Start Loop

  1. Read docs/governance/0002-source-of-truth.html.
  2. Read docs/strategy/0001-phase-1-strategy.html.
  3. Read docs/strategy/0002-phase-1-artifact-contract.html.
  4. Read the latest relevant history entry, currently docs/history/0008-douglass-phase1-v3-page-furniture.html.
  5. Check git status --short before editing.
  6. Append an observability event when beginning meaningful work.

During Work

  1. Protect Phase 1 cleanliness above all downstream ambitions.
  2. Keep raw extraction, cleaned text, candidate objects, future canonical objects, and audit artifacts separate.
  3. Do not introduce retrieval, embeddings, claims, or graph work until the current Phase 1 gate allows it.
  4. Use deterministic methods first. Models can review bounded flagged cases later, but must not become the extractor of record.
  5. Keep generated run artifacts under data/runs/ and out of git unless curated into docs.

Closure Loop

  1. Run the smallest relevant verification command.
  2. Update history when a method, result, failure, or decision matters.
  3. Update lessons when a reusable rule emerges.
  4. Update strategy when the plan or artifact contract changes.
  5. Append an observability event summarizing the outcome.
  6. Stage, commit, and push meaningful durable changes.

Observability Port Rule

Use one observability endpoint: http://127.0.0.1:8765. Do not open new ports for each build. If the dashboard is already running, reuse it. If code changes require a restart, stop the existing process and restart the same port.

When Unsure

If a requested task pulls attention away from clean extraction, pause and explain the tradeoff. The default next action should improve Phase 1 evidence, auditability, or object quality.
