Initial Lessons

Date: 2026-05-27

Lessons Carried Forward

  1. Do not treat automation as intelligence. Perfect the separate steps first.
  2. Do not let retrieval compensate for dirty extraction. It cannot fully recover from contaminated source text.
  3. Do not let graph construction make uncertain extraction look authoritative.
  4. Keep canonical document content separate from retrieval packages.
  5. Use deterministic tools before model calls whenever possible.
  6. Match model tier to task risk and record why a model was chosen.
  7. Use one-book pressure before broadening to many books.
  8. Record decisions, failures, and lessons as first-class project artifacts.
  9. Use candidate-first naming in Phase 1 so clean-looking extraction is not mistaken for canonical truth.
  10. Separate paragraph candidates from structure, page artifacts, and unknown objects before any retrieval work.

Lessons Not Carried Forward

  1. Do not start with many overseers or agent theory.
  2. Do not begin with a broad reusable platform before one book is clean.
  3. Do not rely on Markdown as the long-term documentation layer for this version.
  4. Do not accept a candidate diagnostic as document truth without audit gates.
  5. Do not ask AI to rebuild or fill book content that is not backed by extracted evidence.
