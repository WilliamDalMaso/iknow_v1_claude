Page Image Witness Audit

Date: 2026-05-27

Purpose

The audit now includes visual page evidence. This lets a reviewer compare the rendered PDF page against extracted objects, detector bucket, override bucket, and final candidate bucket before trusting any Phase 1 classification.

Page images are visual witnesses only. They are not OCR, not AI review, not retrieval input, and not canonical promotion.

What Changed

  1. Phase 1 renders each PDF page into data/runs/<book_id>/<run_id>/page_images/.
  2. Image filenames are stable: page_0001.jpg, page_0002.jpg, and so on.
  3. Each page inspection section in phase1_audit.html now shows the rendered page beside object cards.
  4. The observability server can serve generated page images under the existing fixed port.
  5. Validation checks page image count and audit image links.

Verification

  1. Douglass phase1_v3 regenerated successfully.
  2. Rendered page images: 126.
  3. validation_report.json returned pass.
  4. Browser verification loaded page_images/page_0001.jpg with nonzero dimensions.
  5. Existing curated review overrides still applied: 4.

What Remains Weak

Object bounding boxes are listed as text, but the audit does not yet draw visual overlays on the page image. The next possible trust improvement is an overlay view that highlights the source object location on the rendered page.
