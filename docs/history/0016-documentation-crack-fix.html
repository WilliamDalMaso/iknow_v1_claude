Documentation Crack Fix

Date: 2026-05-27

Purpose

This pause checked whether the project was still honoring the original direction: Phase 1 first, evidence before intelligence, HTML documentation as durable memory, one book before automation, and no canonical promotion before audit.

Result: the project is still aligned, but the documentation needed tightening so future work does not inherit stale v1/v2 language or forget the visual evidence and review layers.

Cracks Fixed

  1. Added a Phase 1 Evidence Stack section to the Source of Truth.
  2. Changed the artifact contract from version-specific V2 Acceptance language to current acceptance language.
  3. Updated known limits to reflect the current phase1_v3 audit surface and remaining extraction weaknesses.
  4. Expanded validation wording to include page images, overlay data, and review override checks.
  5. Added audit artifacts and evidence-stack reminders to machine-readable Codex guidance.
  6. Tightened the work loop so observability events are logged before meaningful edits and after verification.

What Did Not Change

  1. No extraction logic changed.
  2. No OCR was added.
  3. No AI/model review was added.
  4. No retrieval, embeddings, graph work, or canonical promotion was added.

Remaining Weakness

The audit and governance layers are stronger than the extractor itself. Paragraph purity, footnotes, captions, tables, figures, front matter boundaries, and section nesting still need evidence-bound improvement before any downstream intelligence should depend on the book.
