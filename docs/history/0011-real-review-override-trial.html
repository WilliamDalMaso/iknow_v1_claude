Real Review Override Trial

Date: 2026-05-27

Purpose

This trial proved the review loop with real Douglass phase1_v3 objects. The goal was not to review the whole book. The goal was to confirm that a small number of audited corrections can move candidate buckets while preserving the original detector decision.

The override file corrects candidate assignment only. It does not create canonical content, delete evidence, or change detector rules.

Overrides Applied

Object Original Bucket Corrected Bucket Reason
douglass_narrative:p0003:obj012 page_artifact_candidate structure_candidate Title-page publication year 1845 is source metadata, not a running page number.
douglass_narrative:p0004:obj005 main_paragraph_candidate structure_candidate Book title, author, and first-publication metadata should not enter the main paragraph stream.
douglass_narrative:p0004:obj007 main_paragraph_candidate structure_candidate PDF production note belongs to source/package metadata, not body prose.
douglass_narrative:p0004:obj008 main_paragraph_candidate structure_candidate PDF creator credit completes the production note and should be preserved outside body prose.

Verification Result

  1. Douglass phase1_v3 regenerated successfully.
  2. validation_report.json returned pass.
  3. Applied override rows: 4.
  4. Main paragraph candidates changed from 352 to 349.
  5. Structure candidates changed from 47 to 51.
  6. Page artifact candidates changed from 117 to 116.
  7. Unknown objects stayed at 1.
  8. The audit page shows Applied overrides: 4 and the per-object override reasons.

What Worked

  1. Only reviewed objects moved buckets.
  2. Each moved object preserved its original detector bucket in the stream row.
  3. Override reasons appeared in phase1_audit.html.
  4. Validation now checks required override fields, bucket names, object ids, uniqueness, and original-bucket agreement.
  5. No canonical outputs were produced.

What Remains Weak

  1. The override rows are currently run-local generated artifacts and are not versioned as curated review decisions.
  2. The audit page exposes override reasons, but it does not yet provide an in-browser editor for adding them.
  3. Structure subtypes are still coarse; these metadata corrections all land in structure_candidate.
  4. This was a small proof, not a complete review of front matter, page furniture, or paragraph purity.

Next Correct Move

Decide whether reviewed overrides should remain run-local or be promoted into a tracked review-decision source that can be replayed across regenerated runs. After that, inspect page 4 metadata and front matter more systematically before adding OCR or model review.
