Durable Curated Review Decisions

Date: 2026-05-27

Problem

The review override loop worked, but the first real override rows lived inside generated run output under data/runs/. That directory is intentionally ignored by git, so reviewed decisions would not survive a fresh clone or a clean regeneration.

Durable review decisions live in tracked reviews/. Generated runs only record which curated decisions were replayed.

New Shape

reviews/<book_id>/review_overrides.jsonl
        ↓ replayed during extraction
data/runs/<book_id>/<run_id>/review_overrides_applied.jsonl

What Changed

  1. Added tracked curated overrides at reviews/douglass_narrative/review_overrides.jsonl.
  2. Phase 1 now reads curated overrides from reviews/ instead of from generated run output.
  3. Each run writes replayed rows to review_overrides_applied.jsonl.
  4. The audit page now distinguishes detector bucket, override bucket, and final candidate bucket.
  5. Validation checks required fields, duplicate object ids, object existence, valid buckets, source path, curated source marker, and original-bucket agreement.

First Curated Decisions

The four Douglass trial overrides were promoted from local generated output into the tracked curated review source. They remain candidate-only and do not promote anything to canonical.

Verification

  1. Douglass phase1_v3 regenerated successfully.
  2. validation_report.json returned pass.
  3. Applied override rows: 4.
  4. The generated run contains review_overrides_applied.jsonl.
  5. The audit exposes detector, override, and final candidate buckets.
  6. No OCR, AI, embeddings, retrieval, graph work, or canonical promotion was introduced.

Remaining Weakness

Review decisions are durable now, but they are still edited by hand. A future step can add a small review UI or command that writes valid curated rows without requiring manual JSONL editing.
