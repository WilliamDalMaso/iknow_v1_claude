0001 — Lane Setup, Wrong-Remote Catch & Ownership Transfer

When: 2026-05-28 (commit e900caf) · Tests after: 27 passed

Why

The task was to run iknow_v1_claude as an independent review/proposal lane for the main project, and to make the lane "truly Claude's" — the project had been bootstrapped by Codex and its governance was written as Codex's self-guidance.

What happened — the safety catch

On first inspection, the local repo was in the correct folder but git remote -v showed origin pointing at WilliamDalMaso/iknow_v1 (the original repo), not the Claude copy. Any push would have landed in the wrong repository, violating the hard safety rule. I stopped and reported instead of proceeding.

After confirmation, the remote was repointed and the lane was seeded (option B — push current local main as the initial Claude-lane state):

git remote set-url origin https://github.com/WilliamDalMaso/iknow_v1_claude.git
git ls-remote --heads origin   # empty -> safe to push normally (no force)
git push -u origin main

What I did — ownership transfer

  • Renamed the self-authored blueprint via git mv (history preserved): 0003-codex-self-guidance.html → 0003-claude-self-guidance.html, 0005-codex-work-loop.html → 0005-claude-work-loop.html, codex_guidance.json → claude_guidance.json.
  • Rewrote the identity inside (Codex → Claude), added maintained_by and a lineage note, and renamed the work-loop key to claude_work_loop.
  • Fixed every cross-reference in README.md, docs/index.html, and 0002-source-of-truth.html.
  • Created the first proposal docs under docs/proposals/claude/: project review, refactor plan, observability upgrade plan, second-book readiness plan, consistency risk report.

How — method & deliberate decisions

  • Preserved docs/history/ verbatim. The Codex history (0001–0048) is an immutable audit log; rewriting it would violate the evidence-first method. Only the forward-looking blueprint was re-homed. The only remaining "Codex" mentions are intentional lineage attributions.
  • HTML, not Markdown. The hard rule forbids Markdown for durable docs, so the proposals are HTML even though .md filenames were suggested. (Confirmed by William later: "html is the way now.")
  • Verified no source code referenced the renamed files before renaming.

Result

Remote correctly set to iknow_v1_claude; lane seeded; governance now Claude-owned; 5 proposal docs added. pytest 27 passed; claude_guidance.json valid; git diff --check clean.

Lesson

Always verify git remote -v before any push in this lane — the clone shipped with the wrong origin.
