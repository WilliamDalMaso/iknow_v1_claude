0006 — Phase C: Read-Only Second-Book (Hobbit) Reconnaissance

When: 2026-05-28 (commit 13d61c6) · Tests after: 42 passed

Why

The real prize is "works for any book." Before committing to that engineering, we needed to learn precisely where the Douglass-specific assumptions live. The fastest, lowest-risk way: run the pipeline on a second book read-only and see what breaks.

Reconnaissance, not widening. Candidate-only run, nothing promoted, no policy adopted, no gold created, Douglass untouched, output git-ignored and deleted afterward. This does not violate the "one book passes gates before widening" rule. I noted it deliberately front-runs my own readiness-plan ordering, justified because it is pure read-only diagnosis.

What I did

python3 src/phase1_extract.py data/books/the_hobbit.pdf \
  --book-id the_hobbit --run-id phase1_dryrun_diagnostic   # then deleted

What I found

  • Core extraction generalizes: 270 pages → 1,853 paragraph candidates, page images, canonical candidates — no crash.
  • Validation failed by construction — the suite contains Douglass-specific acceptance checks (a specific paragraph id, a specific page-join, the guarded-policy gates) run against every book.
  • Furniture detection is literally Douglass text (matches "narrative / life of frederick douglass").
  • The Douglass-gold-tuned paragraph policy is applied globally.
  • The Hobbit has verse/songs the paragraph heuristics never saw.
  • No gold set exists for a second book, so quality is unmeasurable.

Result

Phase C is now concretely scoped: the book-specific surface is (a) paragraph policy, (b) furniture patterns, (c) acceptance/validation checks, (d) curated decisions + gold. Everything else generalized. Findings written to second_book_dryrun_findings.html. Docs-only commit; 42 passed; nothing promoted.
