When: 2026-05-28 (commits d40b2b0, 1426fb6, beae78e, 7f178ce, 744a38b, dc39746) · Tests after each: 42 passed

Why

phase1_extract.py was one 7,524-line file with 106 top-level functions — hard to review, maintain, or generalize to new books. With the regression guard from entry 0004 in place, it became safe to restructure. William explicitly authorized the decomposition.

The pattern (applied identically 5 times)

  1. Move a self-contained group of functions verbatim into a new src/phase1/<module>.py.
  2. Re-import those names (constants included) back into phase1_extract.py, so every existing call site is unchanged.
  3. Run the full suite — the regression test re-runs Douglass and confirms the output is byte-equivalent.
  4. Commit that single step.

Pure relocation only — no logic edits in a move commit. One concern per commit.
