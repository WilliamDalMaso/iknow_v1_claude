0003 — Phase D: Claude-Lane Observability (port 8799)

When: 2026-05-28 (commit 3f9269f) · Tests after: 38 passed

Why

William noted that Codex's observability dashboard already uses http://127.0.0.1:8765, and asked for a separate, better one for the Claude lane. The legacy dashboard was also a flat kind/message/details stream with no severity, no run correlation, and no filtering.

What I did

  • Chose a permanent Claude-lane port: 8799 (8765 stays Codex's, so both run at once). Locked it in claude_guidance.json, both governance HTML docs, and the README.
  • Built src/claude_observability_server.py — a dependency-free dashboard with severity colors, run correlation, client-side search, faceted filters (level/actor/book/phase), issues-only, group-by-run, pause, keyboard shortcuts, summary cards, and /api/summary + /api/meta. It gracefully normalizes legacy flat events.
  • Enriched the shared writer src/observe.py with a structured, backward-compatible schema iknow.observe/2: schema, timestamp, level, actor, kind, book_id, run_id, phase, git_commit, message, details (git commit stamped automatically). The old positional append_event(kind, message, details) still works.
  • Added tests/test_claude_observability.py — writer schema, legacy normalization, and aggregate-summary tests (no socket bound).
  • Left the legacy src/observability_server.py untouched.

How — verification

python3 src/claude_observability_server.py --port 8799   # live smoke test
curl /api/meta -> {"port":8799,"schema":"iknow.observe/2"}
curl /api/summary -> normalized 97 existing events cleanly

Result

A thorough, friendly dashboard on its own port; structured run-correlated events; pytest 38 passed. This is most of Phase D; "decision linkage" (pair an event with each adoption-decision artifact) is deferred until the pipeline seam from Phase B exists.

To run it

python3 src/claude_observability_server.py     # http://127.0.0.1:8799
