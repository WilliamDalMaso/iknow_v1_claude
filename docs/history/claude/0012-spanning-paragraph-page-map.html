0012 — "Paragraphs Not Selected" Investigation + Spanning-Paragraph Page Map

When: 2026-05-30 · Mode: observability-only (no extraction/gold/promotion change)

Why

William noticed pages 6 and 7 of the audit showed fewer objects than visible paragraphs ("Page 6: 4 objects · main_paragraph_candidate: 3"; "Page 7: 2 objects · main_paragraph_candidate: 1") and asked whether paragraphs were being dropped.

Finding: no content is dropped. The pages are the Garrison preface, and the "missing" paragraph on page 7 is the second half of a paragraph that starts on page 6 (obj004__xpage__obj002, bbox pages [6, 7]). The cross-page continuation policy keeps a split paragraph whole and attributes it to its start page, so a page that opens mid-paragraph shows fewer "starts." A token-coverage check confirmed every word of the raw text on pages 6 and 7 is captured in some object — 0 missing tokens.

What I did (observability-only)

  • Added page_object_breakdown() + a /api/page-objects endpoint and a "Per-page object map" panel to the Claude dashboard (src/claude_observability_server.py, port 8799). For each page it shows objects that start there (with "→ continues to p.X" when they span) and objects "↩ continued from p.X" that flow in from an earlier page.
  • Added a contract test (tests/test_claude_observability.py); suite 45 passed.

This did not touch the production audit renderer (build_audit_html in phase1_extract.py); per the standing rule, modifying that production view would need explicit authorization. The dashboard panel is purely additive and reads existing run artifacts.

Verification

Live: page 6 → 4 starts / 0 continued; page 7 → 2 starts / 1 continued-from-p6 — exactly explaining the counts. No extraction artifact changed; the regression baseline is untouched.
