0004 — Phase B: Golden-Output Regression Guard + Determinism Proof

When: 2026-05-28 (commit 5e7b775) · Tests after: 42 passed

Why

Before restructuring the 7,524-line phase1_extract.py, we needed a safety net that proves a change did not alter the book's output. That requires first proving the pipeline is deterministic, then locking the result.

What I did — the investigation

  • Discovered artifacts embed created_at: utc_now() timestamps, so a re-run is not byte-identical. A naive hash regression would flag every JSON. The test had to compare normalized content.
  • Ran Douglass twice into throwaway run-ids and compared. After normalizing volatile fields, two independent runs were identical — the pipeline is deterministic given the PDF. The only run-to-run variation: timestamps, run_id, and absolute paths.
  • Added a normalized content fingerprint to baseline_manifest.py (strip volatile JSON keys; neutralize timestamps/run_id/paths in text; byte-hash images) with CLI fingerprint / verify-fingerprint.
  • Committed normalized_manifest.json as the golden baseline (built from a clean run).
  • Added tests/test_phase1_regression.py — re-runs Douglass (~8s) and asserts it reproduces the fingerprint; skips cleanly when the git-ignored PDF is absent (so CI stays green).

What I found along the way

  • A stale artifact: review_overrides.jsonl sits in the original run dir but is not produced by a current run (recorded as R8).
  • I deliberately did not overwrite the original phase1_v3 run (it is the Codex-produced, git-ignored snapshot). Phase A's byte-manifest stays its record; the Phase B fingerprint (from current code) is the regression baseline.

Result

The pipeline is proven deterministic, and any future drift is now caught automatically. pytest 42 passed. This unlocked the decomposition (entry 0005).

python3 src/baseline_manifest.py verify-fingerprint \
