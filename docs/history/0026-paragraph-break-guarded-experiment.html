Paragraph Break Guarded Experiment

Date: 2026-05-27

Purpose

The taxonomy suggested that the dominant merge failure pattern was merged_across_paragraph_break. This step tested a narrower experimental policy, v2_paragraph_break_guarded, instead of another broad span guard.

This remains experiment-only. The active canonical extraction policy is still v1_consecutive_lines. No OCR, AI review, embeddings, retrieval, graph work, user-facing answers, or structure promotion was added.

Policy Tested

The experimental policy requires paragraph-break evidence such as previous line terminal punctuation, next line capitalization or paragraph-like opening, vertical gap evidence, and indentation or x-position shift. It avoids splitting inside obvious continuous sentences.

Outcome

The experiment was not adopted because it did not reduce likely true accidental merges or merged-across-paragraph-break counts, and it introduced possible over-splitting risk. This is a better result than silently adopting a plausible rule that does not improve measured extraction quality.

Next Step

The sampled rows need visual inspection before another merge rule is attempted. The next correction should likely use stronger evidence from the visual page and source-line sequence, not only punctuation, capitalization, gap, and indentation heuristics.
