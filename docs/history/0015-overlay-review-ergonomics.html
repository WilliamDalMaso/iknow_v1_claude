Overlay Review Ergonomics

Date: 2026-05-27

Purpose

Bounding-box overlays made visual provenance inspectable, but dense pages can become noisy. This step added audit-only controls so a reviewer can focus on specific buckets, override cases, and selected objects without changing extraction logic.

These controls are audit UX only. They do not change classification, candidate streams, curated overrides, OCR, AI review, retrieval, graph work, or canonical promotion.

What Changed

  1. Added overlay bucket toggles for paragraph, structure, page artifact, unknown, and overridden objects.
  2. Added show all, hide all, and overrides-only overlay actions.
  3. Added selected-object details beside the page image.
  4. Added a simple page-image zoom toggle for closer visual inspection.
  5. Existing card filters now continue to hide matching overlay boxes.

Next Possible Improvement

The audit can now focus visual evidence, but it still does not provide a persistent human review UI for writing curated overrides. That should remain a review-governance step, not an extraction or intelligence step.
