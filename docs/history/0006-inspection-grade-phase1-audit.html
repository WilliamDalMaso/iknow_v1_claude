Inspection Grade Phase 1 Audit

Date: 2026-05-27

Why This Step Happened

After the v2 candidate streams worked, the next risk was false trust. Counts and samples are not enough. To make Phase 1 reusable, the audit must expose whether each extraction decision is right or wrong at the page/object level.

The reusable extractor should be trusted because it proves where cleaned content came from, not because the output looks clean.

What Changed

  1. phase1_audit.html now includes a full page inspection index.
  2. Each PDF page has an expandable inspection section.
  3. Each extracted object shows raw evidence beside candidate assignment evidence.
  4. The object view includes raw text, clean text, bucket, confidence, warnings, source lines, classification reasons, and bounding box evidence.
  5. The observability dashboard continues to link directly to the generated audit page.

Verification

  1. The Douglass phase1_v2 run regenerated successfully.
  2. validation_report.json returned pass.
  3. Direct no-cache Phase 1 test execution passed 7 checks.
  4. The in-app browser verified 126 page sections and 517 object cards.
  5. The first page expanded and displayed both Raw Extracted Object and Candidate Assignment.

What Went Right

  1. The audit now supports the real question: was this object classified correctly?
  2. Generated HTML remains local, standard, and self-contained.
  3. No OCR, AI, retrieval, embedding, or graph work was introduced.
  4. The fixed observability port stayed at 127.0.0.1:8765.

Still Weak

The audit is now inspectable, but it does not yet record human corrections or promote candidates.
  1. The page does not yet show rendered PDF page images beside extraction objects.
  2. There is no correction workflow for marking a bucket as wrong.
  3. Repeated running headers are still probably under-classified as structure candidates.
  4. The audit page is larger because it includes all object cards; this is acceptable for one-book trials but should be watched.

Next Correct Move

Use the inspection page to identify the first class of systematic mistakes. The likely next target is repeated headers/page furniture, because v2 detected only one page artifact candidate.
