Narrow Grouping Correction Design

Date: 2026-05-28

Purpose

The project has one confirmed visual-review grouping defect: cp_000103 on page 109. This checkpoint designed a narrow correction for that defect class without changing active extraction behavior.

This is design-only. It did not change extraction behavior, active policy, canonical promotion, OCR, AI/model review, embeddings, retrieval, graph work, structure promotion, or downstream use.

Confirmed Defect

  1. Canonical paragraph: cp_000103.
  2. Affected pages: 107-109.
  3. Gold row: douglass_gold_p0107_0109_001.
  4. Current behavior: pages 107-108 are joined, but page 109 remains a separate continuation fragment.
  5. Expected behavior: one logical paragraph spanning pages 107, 108, and 109.

Why Current Policy Failed

v2_cross_page_continuation can create a two-page joined candidate, but it does not re-evaluate that newly joined candidate against the next page. In this case, the page 107-108 candidate still ends with the incomplete phrase I now had three weeks during, and page 109 begins with continuation text.

Designed Correction

The proposed next experiment is v3_chained_cross_page_continuation_design: after a cross-page join, re-evaluate the newly joined paragraph against the next page's first body paragraph only when the joined text still ends syntactically incomplete and no structure boundary intervenes.

Adoption Requirements

  1. Implement as experiment-only first.
  2. Improve the page 107-109 gold over-split without introducing over-merges.
  3. Keep over-merged gold paragraphs at 0.
  4. List and review all proposed chained joins before adoption.
  5. Record a formal adoption decision before any active policy change.
Downstream remains blocked. The next implementation should test the designed chained-continuation rule as an experiment, not adopt it directly.
