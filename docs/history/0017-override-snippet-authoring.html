Override Snippet Authoring

Date: 2026-05-27

Purpose

Durable curated overrides work, but manually writing JSONL rows is error-prone. This step added copy-ready override templates to the Phase 1 audit so a reviewer can select an object, inspect evidence, and draft a valid review row without browser write-back.

The audit still does not apply corrections. Review rows must be copied into reviews/<book_id>/review_overrides.jsonl, rerun through Phase 1, and pass validation.

What Changed

  1. Selected object details now include a copy-ready JSONL override template.
  2. The template includes object id, page, detector bucket, corrected-bucket placeholder, reason placeholder, reviewer, date, and evidence reference.
  3. The artifact contract now lists override templates as an audit authoring feature.
  4. Validation now checks that the audit contains the override template generator.

What Did Not Change

  1. No server-side write-back was added.
  2. No review validation was weakened.
  3. No extraction classification logic changed.
  4. No OCR, AI review, embeddings, retrieval, graph work, or canonical promotion was added.

Remaining Weakness

A reviewer still has to choose the corrected bucket and explain the reason carefully. That is intentional for now because review corrections should remain deliberate, evidence-bound, and visible in git.
