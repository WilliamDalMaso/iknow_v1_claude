Bounding-Box Overlay Audit

Date: 2026-05-27

Purpose

The audit now draws extracted object bounding boxes over the rendered PDF page. This lets a reviewer inspect whether an object came from the visual location claimed by the extractor.

Bounding-box overlays are evidence only. They do not change classification, do not add OCR, do not add AI review, and do not promote any candidate output to canonical.

What Changed

  1. Line records now preserve x1 in addition to x0, top, and bottom.
  2. Paragraph object boxes merge source-line extents into a single object rectangle.
  3. The audit renders one overlay box per object with a complete bounding box.
  4. Overlay colors distinguish paragraph, structure, page artifact, unknown, and overridden objects.
  5. Object cards and overlay boxes share object ids; click or hover highlights both sides.

Verification

  5. Browser smoke check confirmed clicking a box highlights the matching object card.
  6. Existing curated review overrides still applied: 4.

What Remains Weak

The overlay shows object rectangles, but it does not yet support dense-page controls such as toggling a single bucket, hiding all non-selected boxes, or zooming into a page region. Those are audit ergonomics improvements, not extraction changes.
