Douglass Phase 1 v0 Start

Date: 2026-05-27

Book Choice

The first book is douglass_narrative.pdf. It was chosen over the_hobbit.pdf because it is smaller, better suited to public-domain experimentation, and already exposes real extraction issues without forcing v0 to solve a larger illustrated copyrighted book first.

What Was Built

Phase 1 v0 now has a deterministic extractor at src/phase1_extract.py and a formal artifact contract at docs/strategy/0002-phase-1-artifact-contract.html.

Method Used

  1. Use pdfplumber as the deterministic extractor of record for v0.
  2. Compute a source manifest with file size, page count, SHA-256 hash, run id, and extraction tooling.
  3. Build a page inventory before any cleanup so every page is visible, including image-only pages.
  4. Preserve raw page text unchanged in raw_pages.jsonl.
  5. Create line-level layout objects as a provisional object model.
  6. Apply only minimal text cleanup: right-strip, whitespace collapse, and noise flagging.
  7. Write every cleanup operation to cleanup_log.jsonl.
  8. Generate a local HTML audit page for human review.

Model Use Decision

No LLM was used for the v0 extraction. That was intentional. At this stage, a model would add interpretation before the deterministic evidence layer is stable. A model can become useful later as a reviewer for flagged pages, ambiguous object classification, or OCR repair suggestions, but it should not be the extractor of record.

What The Run Exposed

  1. Page 1 contains visible (cid:...) extraction noise in the title/front-matter area.
  2. Page 2 has no extracted text and one image, so it needs image/OCR-aware review later.
  3. The v0 object model is line-based. That is useful for auditability but not yet a clean paragraph/object model.
  4. Heading detection is intentionally provisional and likely over-detects short uppercase lines.

What Went Right

  1. The source PDF was fully accounted for: 126 inventory rows for 126 PDF pages.
  2. Raw and clean layers are separate, so cleanup is not hiding the original extraction.
  3. The first real extraction defects are visible immediately rather than buried downstream.
  4. The generated audit is local HTML and follows the project formatting standard.

What Is Still Weak

  1. Line-level objects are too primitive for final clean extraction.
  2. Paragraph reconstruction has not started yet.
  3. Heading detection is heuristic and should not be treated as canonical structure.
  4. Image-only page handling only flags the issue; it does not yet OCR or visually describe the page.
  5. The audit summarizes the first pages and flags, but it does not yet provide page-by-page navigation.
This is the right failure shape for v0: we have a complete page inventory and visible risks, not a silent claim that extraction is clean.

Next Improvement

The next Phase 1 slice should improve object segmentation. Specifically, it should merge normal prose lines into paragraph objects while preserving headings, page artifacts, image-only pages, and flagged noisy objects separately.
